6. How the title tag is fetched to head and h1 is fetched to body when rendering a HTML file which has no structure?

Answer: When rendering an HTML file with no structure, the browser automatically places the title tag into the head and the h1 tag into the body based on HTML parsing rules and defaults.

7. Why there is three dots in if...else and do...while ?

Answer: The three dots (...) in if...else and do...while in documentation or examples are placeholders used to indicate omitted code or logic. They imply that additional statements or code blocks should be added in those places.

8. What is the difference between spread and rest operator ?

Answer: The spread operator (...) expands an array or object, while the rest operator (...) collects multiple elements into an array or parameters and used as function parameter.

9. In for in loop while traversing through an object why the output is undefined when accessing using dot operator ?

Answer: The output is undefined because obj.key is looking for a property literally named "key" in the object, instead of dynamically accessing the property using the value of the key variable.

10. Why Components should start with Capital letter?

Answer: Components should start with a capital letter because React uses capitalized names to differentiate components from standard HTML tags.

10. While exporting a component why default keyword is mandatory?

Answer: The default keyword is not mandatory; it specifies a default export, allowing the component to be imported without curly braces, unlike named exports which require them.

12. What is linting error?
